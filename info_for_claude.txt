OVERVIEW
--------
Flask web app where school staff predict match scores for the 2026 FIFA World
Cup. Users earn points for correct predictions and compete on a leaderboard.
Tournament runs June 12 – July 19, 2026.


TECH STACK
----------
Backend:
  - Python 3.12
  - Flask (web framework)
  - SQLAlchemy (ORM)
  - SQLite (database, file: instance/worldcup2026.db)
  - Flask-Login (session management)
  - Flask-Bcrypt (password hashing)
  - python-dotenv

Frontend:
  - Jinja2 templates
  - Bootstrap 5.3.3
  - Bootstrap Icons 1.11.3
  - Vanilla JS (countdown timer only)

Dependencies: flask, flask-sqlalchemy, flask-login, flask-bcrypt, python-dotenv

DATABASE MODELS
---------------
User (table: users)
  - id, name, email (unique), password_hash
  - favorite_team, is_admin (bool), is_approved (bool), created_at
  - Computed props: total_points, exact_scores, correct_outcomes
  - Relationship: has many Predictions

Match (table: matches)
  - id, stage (group/r32/r16/qf/sf/3rd/final), group_label (A-L)
  - team_home, team_away, score_home, score_away (nullable)
  - match_date, is_open (bool), locked_at
  - Computed props: has_result, result_str
  - Relationship: has many Predictions

Prediction (table: predictions)
  - id, user_id (FK), match_id (FK)
  - pred_home, pred_away (user's predicted scores)
  - points_awarded (nullable until result entered)
  - submitted_at
  - Unique constraint: (user_id, match_id)


ROUTES
------
Auth Blueprint (/auth):
  GET/POST /auth/register         New user registration
  GET/POST /auth/login            Login (supports "remember me")
  GET/POST /auth/change-password  Change password (login required)
  GET      /auth/logout           Logout

User Blueprint (/):
  GET  /                          Landing page (countdown, top 5)
  GET  /dashboard                 User home (next 10 open matches, stats)
  GET  /predict                   List open matches grouped by stage
  POST /predict/<match_id>        Submit or update a prediction
  GET  /my-predictions            User's prediction history with points
  GET  /leaderboard               Full rankings table
  GET  /schedule                  All matches; user's predictions overlaid

Admin Blueprint (/admin):
  GET      /admin/                         Stats dashboard
  GET      /admin/users                    User list
  POST     /admin/users/<id>/toggle-approved  Enable/disable user
  POST     /admin/users/<id>/toggle-admin     Promote/demote admin
  POST     /admin/users/<id>/delete           Delete user
  GET      /admin/matches                  Match list + result entry
  POST     /admin/matches/<id>/set-result  Enter result (auto-awards points)
  POST     /admin/matches/<id>/toggle-open Open/close predictions window
  POST     /admin/matches/<id>/update-teams Update TBD team names
  GET      /admin/matches/open-stage?stage=X  Bulk open by stage
  GET      /admin/predictions              View all predictions (filterable)


AUTHORIZATION TIERS
-------------------
1. Unauthenticated  — Landing page only
2. Authenticated    — All user routes accessible
3. Approved user    — Required flag (is_approved=True) to use features
4. Admin            — Full admin panel access (is_admin=True)

Guards:
  - @login_required          Flask-Login decorator
  - require_admin()          checks is_authenticated AND is_admin
  - _require_approved()      blocks is_approved=False users
  - Admins cannot modify their own admin/approved status (self-protection)


SCORING SYSTEM
--------------
calculate_points(pred_home, pred_away, score_home, score_away):
  3 pts — exact score match
  1 pt  — correct outcome (win/draw direction) but wrong score
  0 pts — wrong outcome

Leaderboard sort: total_points DESC, then exact_scores DESC (tiebreaker)
